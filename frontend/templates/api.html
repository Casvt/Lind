Info about the api of Lind
Lind has an API that you can make use of. It has the same functionality as the web-ui. All api-endpoints start with /api.
The API has a rate limiter. A consumer of the API is allowed unlimited requests per minute to all endpoints except the following: POST /api/create. For that/those endpoint(s), there is a rate limit of {{rate_limit}} requests per minute.

POST /api/create

Rate Limit

Use this endpoint to create new Linds. Supply the required information in JSON format in the body of the request. Give optional keys if you want to set them, otherwise leave them out of the body. As the response, you'll get the id of the newly created Lind.

Body:
{
    'url': 'https://example.com',
    'expiration_time': '2022-12-31T23:59',
    'limit_usage': 100,
    'access_password': 'IWantAccess',
    'admin_password': 'IAmTheBoss'
}
  • url (required): the url to shorten
  • expiration_time: the time after which the Lind expires in ISO-8601 format
  • limit_usage: the amount of times a Lind can be used before it expires
  • access_password: the password to access the Lind
  • admin_password: the password to enable and allow managing the Lind

GET /api/<lind_id>

Manage Only

Use this endpoint to get info about a Lind. Replace <lind_id> with the id of the Lind in the url. You need to supply the admin password as an argument in the url. As the response, you'll get info about the Lind. The access_password and admin_password are left out as they are hashed.

Url parameters:
    admin_password=abc
  • admin_password (required): the admin password to manage the Lind

PUT /api/<lind_id>

Manage Only

Use this endpoint to edit info about a Lind. Replace <lind_id> with the id of the Lind in the url. You need to supply the admin password as an argument in the url. Supply the new values in JSON format in the body of the request. Give the keys and values of the fields that you want to edit, otherwise leave them out of the body. As the response, you'll get the new info about the Lind. The access_password and admin_password are not shown as they are hashed, but are open to be changed by supplying a new value in the body.

Url parameters:
    admin_password=abc

Body:
{
    'url': 'https://example.com',
    'expiration_time': '2022-12-31T23:59',
    'limit_usage': 100,
    'access_password': 'IWantAccess',
    'admin_password': 'IAmTheBoss'
}
  • admin_password (required): the admin password to manage the Lind
